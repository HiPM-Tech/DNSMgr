# 待办功能实现方案

## 1. 主题自定义 （ × 未做 ）

**方案：** 前端 CSS 变量 + 用户设置持久化

- 用 CSS 变量定义颜色系统（主色、背景、文字等）
- 支持深浅色模式切换，支持跟随系统
- 支持自定义主色调（颜色选择器）
- 配置存储在用户设置表，多端同步
- 主要是前端工作，实现成本低

---

## 2. 容灾切换 （ × 未做 ）

**方案：** 后台定时健康检查 + 自动切换 DNS 解析

- 新增"容灾配置"数据表，存储：主 IP、备用 IP 列表、检查方式、检查间隔、端口等
- 后台 cron 任务定期对主 IP 做健康检查（HTTP ping / TCP ping）
- 检测到主 IP 不可达时，自动调用对应 DNS 提供商 API 切换到备用 IP
- 恢复后可选择自动切回或手动切回
- 所有切换操作记录审计日志并发送通知

---

## 3. CNAME 拉平 （ × 未做 ）

**方案：** 冲突检测 + 提供商能力适配

- 在保存记录时检测根域（@）或 MX 记录域名上的 CNAME 冲突
- 对支持 CNAME Flattening 的提供商（如 Cloudflare），调用其特定 API 实现
- 对不支持的提供商，给出警告提示，建议改用 A/AAAA 记录
- 前端在编辑记录时实时提示冲突风险

---

## 4. 多线路解析 （ × 未做 ）

**方案：** 各 provider 扩展 line 字段 + 前端动态展示

- 在 DNS 记录类型中增加可选的 `line`（线路）字段
- 每个 provider 实现里扩展线路的读写支持，并声明该提供商支持的线路列表
- 前端记录编辑界面根据当前提供商动态展示可选线路
- 不支持多线路的提供商不显示该字段

---

## 5. 更安全的账号管理（2FA / Passkey） （ × 未做 ）

**方案：** 用户登录后，配置一种或多种验证方式，登录时任选其一

### TOTP 2FA
- 使用 `otplib` 生成密钥，前端展示二维码（`qrcode` 库）
- 登录流程增加 OTP 校验步骤
- 支持备用恢复码

### Passkey（WebAuthn）
- 服务端使用 `@simplewebauthn/server`
- 前端使用 `@simplewebauthn/browser`
- 新增 `credentials` 表存储公钥凭据
- 支持多设备注册

---

## 6. 一键添加邮件解析 （ × 未做 ）

**方案：** 预设邮件服务商模板 + 批量添加

- 内置主流邮件服务商的 DNS 记录模板（Gmail、Outlook、Zoho、Fastmail 等）
- 模板包含：MX、SPF（TXT）、DKIM（TXT）、DMARC（TXT）
- 用户选择服务商后，预览所有将要添加的记录
- 确认后批量调用 API 添加，展示添加结果
- 支持检测已有记录冲突

---

## 7. 团队账号异常行为邮件推送 （ × 未做 ）

**方案：** 审计日志规则引擎 + SMTP 告警

- 在审计日志服务中加入异常检测规则，例如：
  - 短时间内大量删除记录
  - 非常用 IP 登录
  - 多次登录失败
  - 非工作时间操作（可配置）
- 触发规则时调用现有 SMTP 服务发送告警邮件给团队管理员
- 规则和通知接收人可在设置页面配置

## 11. 除邮箱外其他通知方式 （ × 未做 ）

## 12. Whois 域名到期查询 （ × 未做 ）

**方案：** 第三方 Whois API + 定期同步

- 调用第三方 Whois API（`whoiser` `whois-api` 库或 `whoisxmlapi.com`）查询域名到期时间
- 定期同步并将到期时间存入域名表
- 在域名列表展示到期时间，临近到期（可配置天数）时高亮提示
- 触发通知（邮件或其他渠道）提醒续费（可以手动配置是否提醒）
